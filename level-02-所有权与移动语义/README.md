# Level 02：所有权与移动语义

## 通关标准

> 本关结束后，你能解释 Rust 所有权三规则的原理，能预测代码中的 Move/Copy 行为，并能向 C++ 开发者清晰阐述 Rust 所有权与 C++ RAII 的本质差异。

## 核心概念速查

| Rust 概念 | 对比 C++ | 对比 Go/Java/TS | 对比 Swift |
|-----------|---------|----------------|-----------|
| 所有权三规则 | RAII + unique_ptr 只能有一个所有者 | GC 自动管理（无所有权概念） | ARC 引用计数（运行时） |
| Move 语义（默认） | `std::move`（显式，之后对象"有效但未定义状态"） | 引用传递（无移动） | 值类型 Copy，引用类型引用计数 |
| Copy trait | 拷贝构造函数（隐式） | 所有对象都是引用语义 | 值类型自动 Copy |
| Clone trait | 显式深拷贝 | 手动实现 clone() | 值类型自动 Copy，引用类型手动 |
| Drop trait | 析构函数 `~T()` | finalize()（不可靠） | deinit（ARC 管理） |

## 所有权三规则（背诵）

1. Rust 中每一个值都有一个**所有者**（owner）
2. 值在任一时刻**有且只有一个**所有者
3. 当所有者离开作用域，值被**丢弃**（drop）

## 核心差异：Move 是默认行为

```
C++:  默认拷贝，需要 std::move 才能移动
Rust: 默认移动，需要 .clone() 才能复制
```

这一差异是整个 Rust 内存安全模型的基石。
