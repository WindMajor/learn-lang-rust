# Level 03：借用与引用

## 通关标准

> 本关结束后，你能解释 Rust 借用检查器的核心规则，能区分 `&T` 与 `&mut T`，能理解 NLL（Non-Lexical Lifetimes），并能向 C/C++ 开发者说明"Rust 编译期杜绝悬垂指针"与"C++ 运行时崩溃"的差异。

## 核心概念速查

| Rust 概念 | 对比 C/C++ | 对比 Go | 对比 TS/Java |
|-----------|-----------|--------|-------------|
| `&T`（不可变引用） | `const T*`（C） | `*T`（函数参数） | 不可变变量别名 |
| `&mut T`（可变引用） | `T*`（普通指针） | `*T`（无区别） | 可变变量别名（无编译期限制） |
| 借用规则 | 无此概念（程序员自律） | 无此概念 | 无此概念 |
| NLL | 无等价物 | 无等价物 | 无等价物 |
| 悬垂引用（编译期禁止） | 悬垂指针（运行时崩溃/UB） | GC 防止（但需运行时） | GC 防止 |

## 借用规则（背诵）

1. 在任意给定时间，**要么** 只能有一个可变引用，**要么** 只能有任意多个不可变引用
2. 引用必须**总是**有效的（不能比被引用者活得更久）

```
&   = 只读借用 = "我可以看，但不能改，而且可能还有别人也在看"
&mut = 读写借用 = "只有我一个人可以读/写，其他人请退后"
```
